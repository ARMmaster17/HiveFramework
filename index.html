What is Hive Framework

The Hive Framework is a light-weight framework upon which you can build your own cluster computing program with no knowledge of how the Hive is set up or the underlying OS. All Hive apps are built in the Hivil language, which is kinda like BASIC (remember that?). The Hive Framework is also completely driven on auto-pilot. Just install it on a server, plug it into your network, and you're done! Hive takes care of connecting to other Hive nodes and syncing its status across the network. It's also intelligent enough to balance the workload if the current node is faster (or slower) than the other nodes in the Hive.

How do I get it?

DISCLAIMER: HIVE FRAMEWORK IS NOT FINISHED. STABILITY IS NOT GUARANTEED. USE AT YOUR OWN RISK.

Click one of the buttons above to download the latest version. Upon finishing, extract the files to your computer. Next install the latest version of Ruby. Then simply cd into your directory and run

ruby ./HiveStart.rb

That's it! If you would like a more configurable setup procedure, check out the book listed below.

System Requirements

Minimum

You probably could go lower, but here's what we tested Hive on while we developed it (operating at a reasonable speed)

  • Intel Pentium III 800 MHz (1 Core)
  • 256 MB DDR RAM
  • SSH Access
  • Debian Linux 7.x i386
  • Latest version of Ruby
  • Ethernet

Recommended

This is what our build server consisted of. This would probably be the best option for most non-enterprise users.

  • Intel Hex Core
  • 8 GB RAM
  • SSH Access
  • Docker
  • Ubuntu 13.04 x86_64
  • Latest version of Ruby
  • Ethernet

Optimum

Here's what we recommend for more serious/enterprise users

  • Intel Xeon (1 CPU, Quad or better)
  • 4 GB RAM
  • SSH/VNC Access
  • Latest version of i686 or x86_64 Linux (Any distro that runs Ruby)
  • Latest version of Ruby
  • Dual Ethernet (Load distribution or fail-safe mode)

*HIVE Framework will also work on Windows and Mac OS X, but you may need to modify your scripts for full functionality.
